# Air-Gapped Predictive Copilot for Secure MPLS Operations
    ## Team Setup & Development Standards

# Project Overview
    The goal of this project is to build a fully offline AI-powered NOC (Network Operations Center) Copilot capable of:

    - Simulating an enterprise MPLS + SD-WAN network
    - Collecting network telemetry
    - Predicting network degradation before SLA violations
    - Explaining failures using an Offline LLM
    - Recommending corrective actions
    - Displaying everything through a professional dashboard
    - Operating completely within an air-gapped environment

# Git Workflow
    Clone Repository    : git clone <repository-url>
    Switch Branch       : git checkout <branch-name>
    Pull Latest Changes : git pull origin main
    Add Files           : git add .
    Commit Changes      : git commit -m "Meaningful commit message"
    Push Changes        : git push origin <branch-name>

# Frontend Standardization

    Create project using: npm create vite@latest frontend -- --template react

    Install dependencies:
        npm install axios
        npm install react-router-dom
        npm install recharts
        npm install react-icons

    Run frontend: npm run dev

# Backend Standardization

    Run backend: uvicorn app.main:app --reload

# Success Criteria

    The system must answer:

    Q1. What is likely to fail next?
    Q2. Why is the risk elevated?
    Q3. What action should be taken before impact occurs?

The solution must:

    - Operate completely offline
    - Predict failures before SLA violations
    - Explain predictions accurately
    - Recommend corrective actions
    - Present results through a dashboard
    - Demonstrate all functionality in the simulated MPLS/SD-WAN environment

Air-Gapped Requirements
    - No cloud APIs
    - No OpenAI API
    - No internet dependency during runtime
    - All models run locally
    - All telemetry remains local
    - Local vector database only
    - Offline inference only
